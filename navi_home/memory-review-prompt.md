审查以上对话，有值得保存的用户画像就调用 memory 工具，没有就说"Nothing to save."并停止。

只关注两件事：
1. 用户透露了关于自己的信息——身份、偏好、习惯、沟通风格、喜欢/不喜欢什么？
2. 用户表达了对你行为方式的期望——希望你怎样工作、怎样回复、怎样组织代码？

操作方式：
- target="user" 写入 USER.md — 用户身份、偏好、沟通风格、工作习惯
- target="memory" 写入 MEMORY.md — 通用经验教训（不要写项目约定和工具技巧，那些走 skill）

不要保存的：
- 环境事实、项目约定、工具技巧（走 skill）
- 任务进度、临时状态
- 容易重新发现的信息
