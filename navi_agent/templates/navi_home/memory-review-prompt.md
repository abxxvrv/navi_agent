审查以上对话，如果有可以复用的方法论或当前项目约定（不用保存太具体的细节）就调用 memory 工具，没有就说"Nothing to save."并停止。删除和增加都应该保持谨慎，尽量不要大段的删除或者添加，保持记忆精简。

保存前先验证"反复出现"这一信号，不要凭单次会话的印象就写入：
- 用 search_session 检索历史会话，确认某个偏好 / 问题 / 方法是否跨会话反复出现，而非本轮偶发。只有反复出现的才值得固化。
- 涉及具体事实（路径、命令、配置、现有记忆条目）时，用 read_file 核实，确保写入内容准确、且不与 MEMORY.md / USER.md 中已有条目重复。
核实不通过或只是一次性现象的，归入"Nothing to save."。

适合存入的类型：工具特性（比如如何更好的搭配使用某些工具、某些工具该如何使用等）、环境事实（比如某些CLI工具的可执行文件位于哪个路径）、通用性方法论（比如仔细看完SKILL再动手、注意不犯某种错误等）。关于特定任务的经验、流程尽量不要放在这里，而是更新到现有SKILL或者创建新的SKILL。

操作方式：
- target="user" 写入 USER.md — 用户身份、偏好、沟通风格、工作习惯
- target="memory" 写入 MEMORY.md — 跨项目通用经验教训、环境事实和工具特性
- 当前项目的项目级事实不归你管，不要尝试写入。

不要保存的：
- 过于具体的细节、关于单次任务的经验、流程，这个适合放进skill中。
- 任务进度、临时状态
- 容易重新发现的信息
- 敏感信息（密码、密钥）
