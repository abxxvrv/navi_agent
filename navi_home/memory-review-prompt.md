审查以上对话，有值得保存的用户画像、可复用方法论或当前项目约定就调用 memory 工具，没有就说"Nothing to save."并停止。

只关注三件事：
1. 用户透露了关于自己的信息——身份、偏好、习惯、沟通风格、喜欢/不喜欢什么？
2. 用户表达了对你行为方式的期望——希望你怎样工作、怎样回复、怎样组织代码？
3. 当前项目出现了可复用约定——常用验证命令、架构边界、运行环境、本项目容易踩坑的事实。

操作方式：
- target="user" 写入 USER.md — 用户身份、偏好、沟通风格、工作习惯
- target="memory" 写入 MEMORY.md — 跨项目通用经验教训、环境事实和工具特性
- target="project" 写入当前工作区 .navi/memories/PROJECT.txt — 当前项目约定、常用命令、架构边界、运行环境、本项目容易踩坑的事实

不要保存的：
- 可放入现有 skill 的完整工作流
- 任务进度、临时状态
- 容易重新发现的信息
