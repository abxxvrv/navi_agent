01 / OVERVIEW

一个 Agent，贯穿完整工程循环

Navi 不只是聊天界面。它把模型、工具、审批、历史、记忆和外部集成组织成可中断、可恢复、可审计的本地执行环境。

CODE

真实修改工作区

读取、搜索、创建和定向修补文件；配合测试命令完成从定位到验证的闭环。

CONTROL

审批与中断

对危险工具调用实施严格、普通或开放审批，并支持在执行过程中取消当前轮次。

CONTEXT

持久会话与记忆

SQLite 会话历史、FTS5 搜索、上下文压缩、长期记忆和项目级记忆共同维持连续性。

EXTEND

技能、插件与 MCP

按需加载专业技能，兼容约定式插件，并连接外部 MCP server 扩展工具边界。

02 / INSTALL

安装与初始化

需要 Python 3.11 或更高版本。安装脚本会准备 CLI，随后由初始化向导创建 Navi 用户目录并配置模型。

安装 Navi

Windows PowerShell
irm https://raw.githubusercontent.com/abxxvrv/navi_agent/gpt/install.ps1 | iex
macOS / Linux
curl -fsSL https://raw.githubusercontent.com/abxxvrv/navi_agent/gpt/install.sh | bash

初始化配置

navi init

创建 ~/.navi，初始化历史数据库、提示词和内置技能，并引导配置主模型与压缩模型。

检查环境

navi doctor

检查安装、配置和运行环境是否可用。

03 / USAGE

基本使用

INTERACTIVE

交互模式

navi

进入持续对话。工具输出、审批和流式回复都在终端中完成。

ONE SHOT

单次任务

navi run "检查工作区修改"

执行一个任务并退出，适合脚本或自动化流程。

WORKSPACE

指定项目

navi --workspace E:\path\to\project

显式设置 Agent 的工作区边界。

RESUME

恢复会话

navi --continue
navi --resume <session_id>

继续最近会话，或恢复指定会话。

审批模式

navi --approval strict   # 所有高风险动作严格确认
navi --approval normal   # 默认平衡模式
navi --approval open     # 开放执行
navi --yolo              # 等价于 open
中断语义：运行期间按 Ctrl + C 请求取消当前轮次；长命令或子 Agent 在前台运行时可按 Ctrl + G 无损转入后台。

常用 Slash Commands

04 / TOOLS

工具系统

模型通过结构化工具调用操作真实环境。工具注册表统一描述参数、审批级别和执行行为，长任务由任务管理器接管。

类别代表工具能力
文件read_file write_file patch_file读取、创建、覆盖和精确修改文本文件
搜索glob grep按文件模式或内容定位代码
命令bash powershell运行非交互式命令，可前台或后台执行
任务get_task_output wait_tasks kill_task统一管理命令、监控与子 Agent
语义lsp查询定义、引用、实现和符号
会话search_session基于 SQLite FTS5 搜索历史消息
扩展agent skill_view memory派生子 Agent、读取技能与管理长期记忆
05 / EXTENSIONS

技能与插件

SKILLS

按需加载的专业工作流

技能位于 ~/.navi/skills/<name>/SKILL.md。系统提示只注入索引，模型在需要时再读取完整说明，减少上下文占用。

PLUGINS

约定式扩展包

插件可提供 skills、commands、agents、hooks、MCP 和 LSP 配置，并兼容 Grok/Claude 的常见目录约定。

# 临时加载一个可信插件
navi --plugin-dir /path/to/plugin
信任边界：项目和外部插件不会自动执行。需要启用并加入可信路径后，其 Agent、Hooks、MCP 或 LSP 才能启动。
06 / CONTINUITY

会话与记忆

Navi 的用户数据集中在 ~/.navi/，形成从短期会话到长期经验的多层上下文。

history.sqlite3会话、消息、FTS5 全文搜索、锚定窗口和压缩 lineage。
memories/全局 MEMORY 与用户画像，跨项目复用稳定偏好和经验。
项目 .navi/项目记忆与索引，跟随当前工作区保存架构决策和项目约定。
agents/子 Agent 元信息与 transcript，用于同一父会话内恢复。
logs/运行日志和诊断信息。
07 / LANGUAGE INTELLIGENCE

LSP 语义查询

Navi 通过 stdio 连接语言服务器，让 Agent 不只依赖字符串搜索，还能理解符号的定义、引用与实现关系。

{
  "python": {
    "command": "pyright-langserver",
    "args": ["--stdio"],
    "extensions": {".py": "python"}
  }
}
操作作用
goToDefinition跳转到符号定义
findReferences查找语义引用
goToImplementation定位接口或抽象方法的实现
documentSymbol查看文件符号结构
workspaceSymbol搜索工作区符号
restart修改项目语言配置后重启服务
08 / ACCESS

网关与 Web UI

WEB

本地网页控制台

navi web
navi web -p 8080 -w ./proj

支持流式输出、工具卡片、审批、中断、模型切换与会话恢复。

WEIXIN

微信网关

navi weixin login
navi weixin start --account <id>

通过 iLink 接收文本、图片、文件、视频和语音路径。

QQ

QQ 机器人

navi qq login
navi qq start --account <id>

支持私聊和群聊，访问控制默认 fail-closed。

SECURITY

显式授权

navi qq allow <user_id> --account <id>
navi weixin allow <user_id> --account <id>

只有白名单用户和群组可以触发 Agent。

09 / CONFIGURATION

模型配置

配置文件位于 ~/.navi/config.json。主模型与压缩模型可以使用不同 provider，并为每个模型声明上下文窗口等能力。

{
  "default_provider": "openai",
  "default_model": "your-model",
  "compression": {
    "provider": "openai",
    "model": "your-model"
  },
  "providers": {
    "openai": {
      "api_key": "...",
      "base_url": "https://api.example.com/v1",
      "models": {
        "your-model": {"context_window": 131072}
      }
    }
  }
}

当前路由支持 OpenAI-compatible provider，也可连接 LM Studio 等本地兼容服务。恢复旧会话时，优先使用该会话保存的 provider 与 model。

10 / ARCHITECTURE

分层架构

CLIprompt UI、流式输出、slash commands 与一轮对话编排。
RUNTIMEAgent loop、工具调度、审批、中断作用域、后台任务与子 Agent。
MODELProvider 路由、流式模型请求和 request-local 中断。
TOOLS内置工具、注册表、审批策略和 UI/runtime 审批桥。
STORAGE历史、记忆、Agent、Goal 和定时任务持久化。
INTEGRATIONSLSP、MCP、Hooks 与插件发现。
GATEWAYWeb、QQ 和微信多入口适配。
核心原则：UI 主线程只负责输入、按键和渲染；阻塞 runtime 与工具在受控线程执行；外部命令使用可终止的 subprocess；每轮执行由独立的中断作用域统一收尾。
